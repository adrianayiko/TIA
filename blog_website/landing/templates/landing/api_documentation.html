API Documentation

1. Blog List API

Endpoint: /blog/apis/

Method: GET

Description: Retrieves a list of all blog posts.

Response: A JSON array of blog post objects.

2. Blog Detail API

Endpoint: /blog/apis/<int:pk>/

Method: GET, PUT, DELETE

Description:

  • GET: Retrieves a single blog post by ID.
  • PUT: Updates a single blog post by ID.
  • DELETE: Deletes a single blog post by ID.

3. Login API

Endpoint: /login/

Method: POST

Description: Authenticates a user and returns an authentication token.

4. Logout API

Endpoint: /logout/

Method: POST

Description: Invalidates the authentication token and logs out the user.

Authentication

The APIs use token-based authentication. To authenticate, send a POST request to the /login/ endpoint with a JSON object containing username and password fields.

Error Handling

The APIs return standard HTTP error codes for errors.

Example Requests

        
        # Get Blog List
        curl -X GET http://localhost:8000/blog/apis/ -H 'Authorization: Token '

        # Get Blog Detail
        curl -X GET http://localhost:8000/blog/apis/1/ -H 'Authorization: Token '

        # Update Blog Detail
        curl -X PUT http://localhost:8000/blog/apis/1/ -H 'Authorization: Token ' -H 'Content-Type: application/json' -d '{"title": "Updated Title", "content": "Updated Content"}'

        # Login
        curl -X POST http://localhost:8000/login/ -H 'Content-Type: application/json' -d '{"username": "testuser", "password": "testpassword"}'

        # Logout
        curl -X POST http://localhost:8000/logout/ -H 'Authorization: Token '
